## 第一次学习言序

建议按以下顺序阅读：

1. [基础语法与类型](/language/basics/)：理解值、绑定、类型标注和表达式。
2. [控制流](/language/control-flow/)：使用条件与循环组织执行顺序。
3. [函数与闭包](/language/functions/)：提取可复用逻辑，并理解词法作用域。
4. [容器与迭代](/language/collections/)：处理列、元组、典、切片和数据遍历。
5. [类与实例](/language/objects/)：在需要状态与协议时建立对象抽象。
6. [模块](/language/modules/)：把多个文卷组织成具有公开边界的包。
7. [错误处理](/language/errors/)：为失败路径补充诊断、捕获和恢复逻辑。
8. [标准库](/language/standard-library/)：使用文件、JSON、CSV、路径、网络等现成能力。

完成前四步即可编写小型数据处理程序；进入模块和错误处理后，再开始组织需要长期维护的项目。

## 工程实践路线

完成语言基础后，依次阅读[命令工具](/tooling/cli-tools/)、[言包工程工具](/tooling/package-manager/)、[应用构建](/tooling/building-apps/)和[VS Code 扩展](/tooling/vscode/)。日常项目应至少形成“解析依赖 → 静态检查 → 格式化 → 规格测试 → YXB 构建”的反馈循环；参与核心实现时再阅读[从源码构建](/tooling/source-build/)。

准备修改编译器、解释器或 VM 时，再进入[运行时架构](/project/architecture/)；[长期路线图](/project/roadmap/)用于理解已完成能力与兼容性边界，不作为初学者的前置材料。

## Web 开发路线

完成函数、容器、类、模块和错误处理后，先读[Web 开发总览](/web/)，再按`言枢 → 言标 → yanxu-html → yanxu-http → 言枢博客`学习。先从配置化应用和模板获得完整体验，再下钻安全节点与协议对象，最后沿参考项目复习命名路由、页面、JSON/言据 API 与静态文件。
