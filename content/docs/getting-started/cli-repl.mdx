## 常用命令

| 命令 | 用途 |
| --- | --- |
| `yanxu 文卷.yx [-- 参数...]` | 执行源码文件，并把`--`后的值交给`环境.参数` |
| `yanxu` | 进入多行交互环境 |
| `yanxu 查 文卷.yx` | 不执行代码，只做静态类型检查 |
| `yanxu 字节 文卷.yx [-- 参数...]` | 编译并使用字节码 VM 执行 |
| `yanxu 格 --写 文卷.yx` | 就地格式化 |
| `yanxu 试 目录` | 发现并执行 `.yx` 规格测试 |
| `yanxu 包 .` | 显示当前 `言序.toml` 包信息 |
| `yanxu 文 模块.yx` | 从`公`声明生成 Markdown API |
| `yanxu 调 文卷.yx` | 执行并显示语句踪迹 |
| `yanxu 调试服务` | 启动供编辑器使用的 DAP 断点调试适配器 |
| `yanxu 基准 10` | 对照树解释器与 VM 的语义和耗时 |
| `yanxu --version` | 显示版本 |
| `yanxu --help` | 显示命令说明 |

例如`yanxu 服务.yx -- --port 8080`会让`环境.参数（）`返回`【--port，8080】`。包入口同样支持`yanxu 包 运行 . -- 参数...`；分隔符本身不会传入程序。

## 交互环境

不带文件运行`yanxu`即可进入 REPL。它会保留先前定义，类、函数、判断和循环可以跨多行输入；`终`闭合后统一执行。表达式语句显示为`⇒ 值`。上/下方向键浏览持久历史，Tab 会根据关键字、内建法与本次会话声明补全，Ctrl-C 取消当前输入。

| 命令 | 用途 |
| --- | --- |
| `:助` / `:help` | 查看帮助 |
| `:退` / `:quit` | 退出 |
| `:清` / `:clear` | 重建解释器并清空上下文 |
| `:载 路径.yx` | 在当前上下文执行文卷 |
| `:史` | 显示最近的持久历史 |
| `:补 前缀` | 列出与 Tab 相同的补全候选 |

REPL 历史默认保存在 `~/.yanxu_history`，可用 `YANXU_HISTORY` 覆盖。REPL 适合快速试验；需要复用或协作的代码应保存为 `.yx` 文卷，并通过[模块](/language/modules/)组织。
