---
title: 言录：结构化日志
description: 输出六级结构化日志，组合上下文、脱敏、错误踪迹、格式化器与处理器。
---

言录（`yanxu-log`）把日志记录、格式化和输出处理分开。应用可以同时写可读控制台、JSON Lines、言据流或自定义目标，并在输出前递归脱敏结构化字段。

## 安装

```sh
yanbao add log --package 言录 --version "^0.1"
```

言包会自动锁定言据依赖。只写控制台时不需要文件权限；写文件时由顶层应用授权目标路径。

## 等级

| 常量 | 数值 | 典型用途 |
| --- | ---: | --- |
| `追踪级` | 10 | 高频执行细节 |
| `调试级` | 20 | 开发诊断 |
| `信息级` | 30 | 正常业务事件 |
| `警告级` | 40 | 可恢复异常或退化 |
| `错误级` | 50 | 当前操作失败 |
| `致命级` | 60 | 进程或关键子系统无法继续 |

记录器与每个处理器都有最低等级。记录先通过记录器阈值，再由各处理器决定是否输出。

## 控制台与文件

```yanxu
引「包:言录」为 言录；

定 文件器 为 言录.JSON文件处理器（
    「logs/应用.jsonl」，
    言录.信息级，
    10485760，
    5
）；

定 日志 为 言录.日志器（
    言录.追踪级，
    「订单服务」，
    「HTTP」，
    【言录.控制台处理器（言录.调试级），文件器】，
    {「环境」：「生产」}
）；

日志.信息于（「订单已创建」，{
    「请求号」：「req-42」，
    「订单号」：「o-100」
}）；
```

`JSON文件处理器` 每条记录写一行 JSON；`言据文件处理器` 写言据流风格。下一行会越过阈值时先轮转，完整记录不会被拆开。备份名为 `路径.1` 到 `路径.N`。

## 上下文、模块和频道

```yanxu
定 请求日志 为 日志
    .子日志器（「创建订单」，{「请求号」：「req-42」}）
    .频道（「业务」）
    .上下文（{「租户」：「north」}）；

请求日志.调试（「开始校验」）；
请求日志.信息于（「创建完成」，{「耗时毫秒」：18}）；
```

子日志器继承处理器、等级和脱敏配置；模块名以点连接。上下文按创建顺序合并，本次调用字段拥有最后覆盖机会。

## 记录结构

每条记录包含：

- `时间戳`：Unix 毫秒；
- `等级` 与 `等级值`；
- `消息`；
- `模块` 与 `频道`；
- `字段`：合并并脱敏后的结构化字段；
- `错误`：使用 `记错误` 时附加的代码、类别、位置和踪迹。

低于最低等级的调用返回空；已输出的调用返回同一记录典，便于测试和二次处理。

## 错误踪迹

```yanxu
试 则
    执行订单（）；
救 所误 则
    日志.记错误（「订单执行失败」，所误，{「订单号」：「o-100」}）；
终
```

不要只记录 `所误.消息`，否则会丢失错误代码、类别、位置与完整调用踪迹。

## 字段脱敏

默认敏感键包含 `密码`、`口令`、`令牌`、`token`、`authorization` 和 `密钥`，匹配后递归替换为 `***`。可按应用词汇调整：

```yanxu
日志.设脱敏（【「密码」，「会话」，「secret」】，「[已遮蔽]」）；
```

脱敏针对结构化字段。消息正文是不可解析文字，因此敏感数据应放在字段中，不要拼接进消息。

## 自定义格式与处理器

```yanxu
法 格式化（记录：典）：文 则
    归 （记录【「等级」】 加 「:」 加 记录【「消息」】）；
终

法 发送（正文：文，记录：典）：空 则
    # 交给应用拥有的队列或传输层
    言 正文；
    归 空；
终

日志.加处理器（言录.函数处理器（言录.警告级，格式化，发送））；
```

内建格式化器为 `控制台格式`、`JSON行格式` 和 `言据格式`。`内存处理器` 提供 `各文本`、`各记录` 与 `清空`，适合测试日志行为。

便捷入口 `新建(最低等级)` 创建单控制台日志器，`静默(最低等级)` 创建无处理器日志器。
