言枢 0.2 保留`yanxu-web`技术包标识和主要 0.1 API，可以逐步升级。

## 更新模块别名

```yanxu
# 旧
引「包:yanxu-web」为 Web；

# 新
引「包:yanxu-web」为 言枢；
```

## 配置化创建

```yanxu
定 配置项 为 言枢.配置（）
    .设模板根（「templates」）
    .设静态根（「static」）；
定 应用 为 言枢.创建应用（配置项）；
```

`创建应用`默认加入安全首部。已有等价中间件的项目应删除重复实现，或调用`设自动安全首部（假）`。

## 命名路由

```yanxu
应用.命名取（「/posts/:id」，「文章详情」，文章）；
定 地址 为 上下文.反向（「文章详情」，{「id」：「first」}）；
```

优先命名页面会链接到的路由，再逐步消除硬编码地址。

## 渐进迁移页面

`HTML响应`继续支持`yanxu-html`节点。新页面可先改为`.yb.html`并使用`应用.模板响应`，旧页面无需一次性重写。

原有`HTML.原始`使用点必须逐项审计；只有可信内容才迁移为`言枢.言标.信任HTML`。

## 测试

旧测试仍可构造`HTTP请求`并调用`应用.处理`。推荐改用：

```yanxu
定 客户端 为 言枢.测试客户端（应用）；
定 响应 为 客户端.取（「/posts/first」）；
```

兼容类型`Web应用`、`Web上下文`和`Web开发服务器`分别继承新的言枢类型，旧类型标注继续有效。

## 跨仓发布顺序

先提交并发布言枢 0.2，再把博客等消费项目的版本要求更新为`^0.2`，最后由言包重新生成锁文件。不要手工猜测 Git 修订或内容校验和。
