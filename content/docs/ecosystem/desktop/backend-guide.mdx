---
title: 贡献新的平台后端
description: 在不泄漏系统句柄或下沉高级控件的前提下扩展言台。
---

首版 Windows、macOS、Wayland 和 X11 共享 winit／softbuffer 后端。新增集成应先扩展公共路径；只有上游确实无法表达所需平台原语时，才增加小型目标适配器。

## 边界

后端可以接触系统窗口、显示器、输入、剪贴板、文件对话框、字体和像素表面，但不能实现按钮、输入框、布局、主题或业务组件，也不能把 HWND、NSWindow、Wayland／X11 对象或原始指针放进 ABI 返回值。

## 实现步骤

1. 在无系统句柄的模型层定义状态和所有权。
2. 先定义事件或绘制协议版本与兼容规则。
3. 在 ABI 边界增加参数、权限、线程和资源类型校验，以及稳定`PLATFORM_*`错误。
4. 必要时增加最小言序包装，并重新生成 API 快照。
5. 把系统状态映射为逻辑像素统一事件，把完整 YXDR 帧映射为后端绘制。
6. 为转换、损坏输入、容量、生命周期和错误路径增加无显示器单元测试。
7. 增加可自动退出的真实窗口集成场景。
8. 更新能力查询、协议文档、第三方许可与六目标 CI。

高频新事件必须选择明确策略：连续状态可保留最新，增量滚轮应累积，按键、按钮、IME、拖放与生命周期事件永不合并。新绘制能力优先增加带自描述长度的次版本操作码；改变既有解释才提升主版本。

完整贡献流程和本地命令见[言台 1.0 后端指南](https://github.com/yanxulang/yanxu-platform/blob/v1.0.0/docs/BACKEND_GUIDE.md)。
