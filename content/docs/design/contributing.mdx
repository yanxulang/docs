---
title: 参与贡献
description: 选择正确仓库，运行对应门禁，并提交范围清晰、可验证的改动。
---

言序采用多仓结构：语言核心、言包、言据、VS Code 扩展、官网和文档站独立版本化。先确认问题属于哪一层；跨仓变化应在描述中列出兼容顺序和发布依赖。

核心改动至少运行：

```sh
cargo fmt --all -- --check
cargo test --locked
```

文档改动至少运行：

```sh
npm run check
npm run build
```

新增语法要同时更新规范、解析与诊断、类型或运行语义、格式化、两条执行路径、兼容语料、编辑器语法和文档示例。新增生态能力则应保留独立版本与生产边界，不把仓库规划误写成核心现状。

提交问题时附最小复现、版本、平台、期望与实际结果；安全问题按各仓库 `SECURITY.md` 的私密渠道处理。
