[`yanxu-http`](https://github.com/yanxulang/yanxu-http)负责服务端协议对象与 TCP 单请求连接。它不提供路由或 HTML 清洗，因而可以独立用于协议测试、教学或其他框架。

```sh
yanbao add http --version '^0.1'
```

## 解析请求

```yanxu
引「包:http」为 HTTP；
引「标准:字节」为 字节；

定 请求 为 HTTP.解析请求（
    「POST /search?q=言序&q=web HTTP/1.1\r\nHost: localhost\r\nContent-Length: 6\r\nCookie: theme=dark」，
    字节.从文字（「正文」）
）；

言 请求.方法；
言 请求.路径；
言 请求.查询全部（「q」）；
言 请求.Cookie值（「theme」）；
言 请求.正文文字（）；
```

`解析请求头`用于没有正文的请求；`解析请求`还会验证实际字节数与`Content-Length`完全一致。查询参数保留重复值，首部查询不区分大小写。

## 严格的 0.1 模型

| 项目 | 行为 |
| --- | --- |
| 版本 | 仅 HTTP/1.1，必须有非空`Host`。 |
| 请求目标 | origin-form路径或`*`。 |
| 首部 | UTF-8、严格 CRLF，不支持折叠。 |
| 正文 | 仅单一合法`Content-Length`。 |
| 默认预算 | 首部 64 KiB、正文 4 MiB、超时 5 秒。 |
| 分块请求 | 出现`Transfer-Encoding`即明确拒绝。 |
| 连接 | 每个连接一个请求，响应固定关闭。 |

这个子集避免在未实现的协议细节上进行宽松猜测。它仍然只是开发与教学基线，不包含 TLS、并发工作池、持久连接、流式正文或可信代理模型。

## 构造响应

```yanxu
定 响应 为 HTTP.JSON响应（{「ok」：真}）；
响应.设首部（「cache-control」，「no-store」）；
响应.添Cookie（
    HTTP.Cookie（「sid」，「abc123」）.设安全（真）
）；
定 线上字节 为 响应.编码（）；
```

响应工厂包括`字节响应`、`文字响应`、`HTML响应`、`JSON响应`、`言据响应`、`状态响应`、`响应字节`和`重定向响应`。编码器统一管理`Content-Length`、`Connection`和`Set-Cookie`，并拒绝首部值中的 NUL、CR、LF。

Cookie 默认`Path=/`、`HttpOnly`、`SameSite=Lax`。开发期`Secure`默认关闭，生产 HTTPS 会话必须显式打开；`SameSite=None`没有`Secure`时会失败。

## 言据的边界

`言据响应`接收已经序列化的文字。先调用`yanju`的`言据.序列化（值）`，再交给 HTTP 层设置`application/vnd.yanxu.yanju; charset=utf-8`。这样数据格式实现不会复制进协议库。

全部公开对象、错误类别与单次 TCP 示例见[`yanxu-http/docs`](https://github.com/yanxulang/yanxu-http/tree/main/docs)。要把请求交给配置、路由、中间件和言标页面，继续阅读[言枢](/web/framework/)。
