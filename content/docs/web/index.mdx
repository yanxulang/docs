言序 Web 栈由三个职责独立、单向依赖的基础库和一个综合示例组成。它们不进入语言核心，而是以独立仓库迭代，使 HTML 安全、HTTP 协议和应用框架可以分别测试、替换与演进。

```text
yanxu-webblog   完整博客 · JSON API · 言据 API · 静态文件
      │
      ▼
yanxu-web       应用 · 路由 · 上下文 · 中间件 · 错误边界 · 开发服务器
   ┌──┴──────────────┐
   ▼                 ▼
yanxu-html       yanxu-http
安全 HTML 节点     HTTP/1.1 请求与响应
```

## 如何选择

| 目标 | 从哪里开始 |
| --- | --- |
| 生成邮件、静态页面或服务端 HTML | 只使用`yanxu-html`。 |
| 学习 HTTP 解析、构造响应或实现自有服务器 | 只使用`yanxu-http`。 |
| 编写带路由和中间件的本地 Web 应用 | 使用`yanxu-web`，同时依赖 HTML 与 HTTP。 |
| 寻找可直接运行和改造的项目 | 克隆`yanxu-webblog`。 |

## 前置知识

建议先掌握[函数与闭包](/language/functions/)、[容器](/language/collections/)、[类与实例](/language/objects/)、[模块](/language/modules/)和[错误处理](/language/errors/)。要理解开发服务器，再阅读[安全套接字](/language/sockets/)。

所有四个仓库当前版本均为`0.1.0`，语言与网络 API 以言序核心`1.1.4-A`和言包`0.1.0`为基线。该组合提供无 submodule 的锁定包模块加载边界。首版强调清晰边界和可执行参考，不把串行开发服务器描述为生产服务器。

## 推荐学习顺序

1. [安全 HTML](/web/html/)：理解普通文字、属性和显式原始内容；
2. [HTTP/1.1 基础](/web/http/)：理解请求解析、响应编码和连接边界；
3. [Web 框架](/web/framework/)：把请求交给路由、中间件和处理器；
4. [完整博客](/web/webblog/)：沿一个真实请求阅读全部四层；
5. [安全与路线图](/web/security-roadmap/)：确认当前部署边界和后续协议方向。

## 取得源码

四个仓库都可以普通克隆；依赖由言包根据清单和锁文件安装：

```sh
git clone https://github.com/yanxulang/yanxu-html.git
git clone https://github.com/yanxulang/yanxu-http.git
git clone https://github.com/yanxulang/yanxu-web.git
git clone https://github.com/yanxulang/yanxu-webblog.git
yanbao --manifest-path yanxu-web install
yanbao --manifest-path yanxu-webblog install
```

`言序.toml`保存 Git 来源与版本要求，`言序.lock`固定精确提交和内容校验。运行时只把经过锁文件验证的包根用于模块加载，不把依赖清单中的网络、环境或进程权限合并到应用；顶层显式声明的同名依赖优先，未声明时再使用依赖包自身的传递依赖。
