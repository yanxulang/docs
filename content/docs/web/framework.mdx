[`yanxu-web`](https://github.com/yanxulang/yanxu-web)同时依赖`yanxu-html`与`yanxu-http`，但不复制二者的职责：HTML 节点负责安全输出，HTTP 对象负责协议字节，框架只做应用编排。

## 创建应用

```yanxu
引「包:yanxu-web」为 Web；
引「包:yanxu-html」为 HTML；

法 首页（上下文）则
    归 Web.HTML响应（HTML.元素（「h1」，【】，【
        HTML.文字（「言序 Web」）
    】））；
终

法 文章（上下文）则
    归 Web.JSON响应（{「id」：上下文.参数值（「id」）}）；
终

定 应用 为 Web.应用（）；
应用.取（「/」，首页）；
应用.取（「/posts/:id」，文章）；
应用.挂静态（「/static」，「static」）；

Web.开发服务器（应用，「127.0.0.1:8080」）.运行（）；
```

路由支持字面路径段、`:名称`单段参数和末尾`*名称`剩余路径参数。路径不存在时返回 404；路径存在但方法不匹配时返回 405，并生成`Allow`首部。

## 请求上下文

处理器接收`Web上下文`，可通过以下成员访问请求数据：

- `参数值（名称）`读取路径参数；
- `查询值（名称）`读取第一个查询值；
- `首部值（名称）`读取请求首部；
- `Cookie值（名称）`读取 Cookie；
- `状态值（名称）`读取应用共享状态。

应用状态通过`应用.状态`典在装配时设置。0.1 开发服务器串行执行，但仍建议把共享状态视为应用级配置，避免处理器无约束修改全局容器。

## 中间件与错误

```yanxu
法 安全首部（上下文，下一步）则
    定 响应 为 下一步（）；
    响应.设首部（「x-content-type-options」，「nosniff」）；
    归 响应；
终

法 错误响应（上下文，所误）则
    归 Web.状态响应（500，「Internal Server Error」）；
终

应用.使用（安全首部）；
应用.设错误处理器（错误响应）；
```

中间件按注册顺序进入，调用`下一步`后逆序返回。它可以在请求前准备上下文，在响应后统一添加首部，或不调用`下一步`直接短路。路由、中间件和处理器错误进入统一错误处理器；每个阶段都必须最终返回`HTTP响应`。

## 响应与静态文件

`HTML响应`和`HTML状态响应`调用`yanxu-html`安全渲染。`JSON响应`、`言据响应`、`文字响应`、`字节响应`与`状态响应`委托给 HTTP 层。

静态文件只接受文件根内的安全相对路径，拒绝绝对路径、反斜杠、NUL、`.`、`..`和空路径段。框架识别常见 Web 媒体类型并默认设置`Cache-Control: no-cache`，适合开发时快速刷新。

## 开发服务器边界

服务器串行接受连接，每条连接只处理一个 HTTP/1.1 请求并关闭。它没有 TLS、并发、长连接、流式请求、生产日志或优雅重启。完整生命周期和扩展点见[`yanxu-web/docs`](https://github.com/yanxulang/yanxu-web/tree/main/docs)；可运行组合见[yanxu-webblog](/web/webblog/)。
