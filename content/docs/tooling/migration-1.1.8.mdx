1.1.8 是兼容性修复版本，不改变语言规范 1、标准库 API、原生 ABI v1/v2、格式 2
清单与锁文件、YXB、Bundle 或嵌入接口。命令行项目和非 Windows 制品可直接替换运行时。

## GUI 项目为什么要升级

Windows MSVC 链接器默认只为可执行文件保留 1 MiB 主线程栈。VM 所有者线程同时驱动
原生事件循环时，正常的控件布局、重绘与言序方法嵌套可能超过该默认值。1.1.8 为
x86_64 与 ARM64 Windows 的`yanxu.exe`保留 8 MiB 栈；GUI Bundle 携带同一运行时。

## 升级步骤

1. 安装言序 1.1.8，并用`yanxu --version`确认版本。
2. 言界项目把清单最低版本改为`言序 = ">=1.1.8"`，再用言包显式更新锁文件。
3. 在对应 Windows 原生目标重新构建 YXB 与 Bundle，不再分发由 1.1.7 打包的旧 Bundle。

无需修改`.yx`源码、ABI v2 调用、原生库摘要或资源声明。自定义嵌入宿主创建的线程不使用
官方可执行文件栈设置，仍应由宿主分配足够栈空间。详细运行模型见
[应用和事件循环](/gui/app-loop/)，平台范围见[平台能力表](/gui/capability-matrix/)。
