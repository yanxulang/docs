言包（`yanbao`）是言序的独立官方包管理器。它维护`言序.toml`，但把依赖解析和`言序.lock`生成交给语言核心的`yanxu::package`，因此命令工具与运行时不会产生两套包格式。

## 安装

首个源码版本可从 GitHub 安装：

```sh
git clone https://github.com/YanXuLang/yanbao.git
cargo install --path yanbao
yanbao --help
```

`yanbao run`还需要可从`PATH`找到`yanxu`。使用自定义构建时可设置`YANXU_BIN=/绝对路径/yanxu`。

## 创建项目

```sh
yanbao init 我的项目
yanbao --manifest-path 我的项目 list
```

`init`创建`言序.toml`、`言序.lock`、`src/主.yx`和`.gitignore`。命令不会自动创建 Git 仓库，也不会覆盖已有模板文件；确需覆盖时显式使用`--force`。

## 管理依赖

```sh
# 中央索引依赖；缺省版本要求为 *
yanbao add 文字工具 --version '^1.2'

# 路径依赖
yanbao add 共享工具 --path ../共享工具 --version '^1'

# Git 依赖
yanbao add 远程工具 --git https://github.com/example/工具.git --rev main

yanbao remove 共享工具
```

`add`和`remove`默认立即重新生成锁文件。若解析失败，言包会同时恢复原清单和原锁文件，避免留下半完成状态。只准备清单而暂不解析时可使用`--no-lock`。

## 安装、更新与离线复现

```sh
yanbao install
yanbao install --offline
yanbao update
yanbao run
yanbao run -- --once
```

`install`优先验证现有锁定和缓存；`update`显式移除旧锁定后重新选择版本；`--offline`禁止网络获取。缓存位置沿用核心运行时，可通过`YANXU_CACHE`覆盖。

1.1.3 的核心解压器会在索引制品写入缓存前逐项校验。符号链接、硬链接、设备文件、管道、越界路径和重复写入均被拒绝；压缩体积、单文件、展开总量、条目数和路径长度受硬上限保护。

路径依赖锁按相对路径、文件长度与内容计算 SHA-256；`.git`、`.yanxu`、`target`、`.DS_Store`和`言序.lock`不参与校验，因而在仓库内安装言序或跨 macOS/Linux/Windows 复现时不会产生无关漂移。其余文件变化会使旧锁失效，应显式运行`yanbao update`或`yanxu 包 更新`。

Git 分支或标签在锁文件中保存为精确提交。`install`优先复用并验证该提交；`update`会重新查询远端，避免把缓存中的陈旧分支写回新锁。已经通过锁定和内容校验的包根自动用于模块加载，不需要 submodule，也不需要把全局缓存路径加入`[权限].文件`。这项授权只允许加载包源码，不会继承依赖清单中的普通文件、网络、监听、环境或进程权限。

## 自动化输出

所有命令接受全局`--json`，输出带`schema_version: 1`的机器可读结果：

```sh
yanbao --manifest-path . install --json
```

## 当前边界

0.1 版本完成本地项目和依赖生命周期，但公开注册表、包搜索、发布、撤回、审计与签名仍属于 1.3 包生态里程碑。当前核心 CLI 的`yanxu 包`、`包 运行`、`包 锁`和`包 更新`继续作为兼容入口；新项目工作流优先使用言包。

源码、问题与贡献说明见[YanXuLang/yanbao](https://github.com/YanXuLang/yanbao)。
