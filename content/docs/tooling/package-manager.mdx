言包（`yanbao`）是言序的官方工程与包管理工具。0.2.0 起，命令层完全由言序源码编写；仓库没有 Cargo 工程或 Rust 业务代码。0.2.1 起，言包通过独立 GitHub Release 发布，正式分发包内置专用言序 1.1.6 运行时。它负责项目初始化、依赖增删、安装更新、测试、构建、缓存、诊断、打包和注册表交互，但不复制第二套 TOML、SemVer、依赖解析、锁图或 YXB 实现。

## 独立安装

普通用户直接使用言包 Release 的跨平台安装脚本；安装包同时校验言包、启动器和专用运行时版本，不要求系统已有`yanxu`：

```sh
curl -fsSL https://raw.githubusercontent.com/YanXuLang/yanbao/main/scripts/install.sh | sh
yanbao --version
yanbao doctor
```

Windows 使用仓库中的 PowerShell 安装器。正式分发的启动器只定位同一安装目录中的`src/主.yx`和专用`yanxu-1.1.6`，所有业务逻辑仍由言序执行。只有从源码仓库开发言包时才需要言序 1.1.6 或更高版本；此时可用`YANXU_BIN=/绝对路径/yanxu`指定开发核心。任何场景都不需要 Rust 版言包，因为它不存在。

## 创建项目

```sh
yanbao init 我的项目 --name 示例
yanbao doctor --manifest-path 我的项目
```

`init`创建格式 2 的`言序.toml`、`言序.lock`、`src/主.yx`和`.gitignore`；不会创建 Git 仓库，也不会无提示覆盖已有模板。中文命令可写作`yanbao 新 我的项目`。

## 管理完整依赖图

```sh
yanbao add 工具 --package 共享工具 --path ../共享工具 --version '^1'
yanbao add HTTP --package yanxu-http --git https://github.com/YanXuLang/yanxu-http.git --rev main
yanbao add 测试工具 --dev --path ../测试工具
yanbao remove 工具
yanbao install --offline
```

`add/remove`默认重新解析完整直接与传递图；失败时同时恢复清单和锁文件。`--no-lock`可只编辑清单。格式 2 锁文件固定每条依赖边、精确版本/提交、来源、校验和、目标、导出和原生制品。

```sh
yanbao tree
yanbao why HTTP
yanbao outdated
yanbao update --dry-run
yanbao doctor
```

这些命令分别解释依赖树、引入路径、可更新项、更新计划和工具链状态。`update --dry-run`不会改写锁文件；正式`update`才重新选择允许范围内的版本。

## 开发、构建和分发

```sh
yanbao check
yanbao test
yanbao run -- --once
yanbao build --release
yanbao build --release --standalone
yanbao doc
yanbao pack
yanbao vendor
yanbao audit
```

`build`调用核心生成 YXB 或当前平台自包含程序；`pack`生成确定性`.yxp`；`vendor`复制完整锁定图以便离线恢复；`audit`复核校验和、来源、许可证、重复版本与原生制品。工作区会对声明成员执行同一套命令。

## 职责与安全边界

言包通过`yanxu package protocol`的版本化 JSON 协议调用核心`yanxu-package`。启动时会协商清单 2、锁 2、YXB 1、工程协议 1 和原生 ABI 1；核心不兼容时会明确要求先升级言序。

路径、Git 和索引包只有通过锁定与内容校验后才可参与模块加载。依赖清单的文件、网络、监听、UDP、进程或原生扩展权限不会传给应用；第三方包也不能自动执行构建脚本。公开注册表在完成安全评审前仍不作为 1.1.6 发布目标。言包 Release、标签和 CI 与核心运行时分别管理，不能以核心 Release 替代言包自身验收。

源码、问题与贡献说明见[YanXuLang/yanbao](https://github.com/YanXuLang/yanbao)。
