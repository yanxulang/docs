---
title: 从 1.1.6 升级到 1.1.7
description: 保持 ABI v1 与旧 YXB 兼容，并采用 GUI、ABI v2 和 Bundle。
---

1.1.7 不改变语言规范 1 语法。ABI v1、旧 YXB v1、Standalone 与嵌入 API 继续兼容；
普通项目升级运行时并重新测试即可。使用新能力的格式 2 项目把最低版本改为
`言序 = ">=1.1.7"`，再用言包 0.5.0 重新生成锁文件。

GUI 项目添加`yanxu-gui ^0.1`、`[应用]`、`[应用.窗口]`与
`图形界面 = true`。剪贴板和文件对话框按需单独申请；图标必须是包内普通资源。
`yanbao init 项目 --gui`可生成完整模板。

异步原生包迁移到 ABI v2 后，应遵守借用参数、模块释放结果、成对保留/释放回调、
后台只 post、UI 只在 VM 所有者线程操作和父资源递归关闭等规则。继续使用 v1 的包不必
改入口或结构布局。

图形分发使用`yanbao build --release --bundle`。Bundle 携带运行时、YXB、锁定后端、
资源、图标和许可证，首次运行不下载。交叉构建必须提供同目标 runtime；六目标产物在
对应原生 runner 编译，不能复制改名。

遇到摘要/目标错误时显式重建锁文件，不绕过校验；遇到权限拒绝时同时检查应用申请和
宿主上限；后台工作通过事件队列返回，不直接操作 VM 或控件。
