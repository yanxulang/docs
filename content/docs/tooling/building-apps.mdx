---
title: YXB 与独立应用
description: 把言序项目编译为不依赖原始源码的确定性应用制品。
---

言序 1.1.5 的“完整编译”是完整包字节码编译，不是 LLVM 或 Cranelift 机器码后端。编译器读取格式 2 锁图，把入口、项目模块、依赖模块、声明资源、权限、目标和调试路径写入 YXB 应用归档。

## 直接使用核心

```sh
yanxu compile . -o build/我的应用.yxb --release
yanxu run build/我的应用.yxb -- 参数甲 参数乙
```

生成后可以把 YXB 复制到临时目录、删除源码副本再运行。YXB 仍需要目标机器安装兼容的言序 VM，但不会重新读取`.yx`、依赖清单或资源目录。

## 使用言包工作流

```sh
yanbao check
yanbao test
yanbao build --release
yanxu run build/我的应用.yxb
```

言包会把制品放入项目`build/`，工作区则按成员逐项执行。`yanbao clean`清理派生制品；增量缓存以内容校验和为键，不把源码时间戳作为语义输入。

## 当前平台自包含程序

```sh
yanbao build --release --standalone
```

该命令把当前执行的言序 VM 与 YXB 封装为一个程序，目标机器不需要另装言序。制品只适用于构建时目标；应在 Windows、Linux、macOS 各自构建，不能跨系统复制代替交叉编译。

## 确定性与发布门禁

- 相同源码、锁图、资源、目标和配置必须生成逐字节相同的 YXB/YXP。
- 构建前必须验证完整锁图和每个包的内容校验和，不会在导入时临时选版。
- 资源只来自`[资源]`声明，归档路径会拒绝绝对路径和父目录越界；程序通过`标准:资源`的读取与目录接口访问。
- 应在删除源码副本后运行 YXB，并在每个支持平台验证独立应用。

YXB 不是原生机器码，也不提供服务端 TLS、HTTP/2/3、异步 I/O 或多线程任务调度。
