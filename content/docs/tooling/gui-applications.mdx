本文保留现有`yanxu-gui`（言窗）路线的快速参考。新项目请先阅读独立的
[图形界面概览](/gui/)、[言窗与言界的区别](/gui/routes/)和[选择 GUI 路线](/gui/choosing/)；
言窗继续兼容，不要求迁移。

言序 1.1.7 不把 GUI 固化进标准库。官方包`yanxu-gui`（言窗）以中文 API 封装
ABI v2 后端`eframe/egui + winit`，支持 Windows、macOS 和 Linux Wayland/X11、
中文 IME、每窗口 DPI、图片、Canvas、剪贴板、文件对话框和基础辅助功能。

## 创建和运行

```sh
yanbao init 我的窗口 --gui
yanbao check --manifest-path 我的窗口
yanbao run --manifest-path 我的窗口
```

```yanxu

定 应用 为 界面.应用（「示例」）；
```

控件含文字、按钮、单/多行输入、复选/单选、下拉、滑块、进度、图片、列表、滚动、
分隔线、标签页、菜单和 Canvas；布局含纵向、横向、网格、层叠和滚动。公开类型描述
窗口配置、样式、颜色、尺寸、事件和对话框结果，不用`任意`掩盖 API。

## 事件、线程和资源

点击、输入、焦点、键盘、鼠标、滚轮、窗口移动/缩放/关闭、拖放、定时器和自定义事件
都变为结构化事件典。原生线程只投递到有界宿主队列，言序回调只在 VM 所有者线程
执行。高频连续事件会合并；窗口关闭按子到父顺序释放控件回调，应用退出后资源计数
归零。定时器最小间隔 10 毫秒，支持单次、周期和取消。

文本编辑直接使用系统 IME 和字体回退，支持中文组合输入、候选框、光标/选择、复制、
剪切、粘贴、撤销/重做、中英文混排、Emoji 与高 DPI。可访问名称、描述、控件角色、
Tab 导航、系统缩放和高对比度基础样式进入 egui 的辅助功能树。

## 权限和 Bundle

`图形界面`不授予文件或网络。剪贴板、文件对话框、通知、托盘、外部地址和全局快捷键
分别授权；应用申请还会与嵌入宿主上限取交集。

```sh
yanbao build --manifest-path 我的窗口 --release --bundle
```

macOS 输出`.app`，Windows 输出 GUI 子系统应用目录，Linux 输出 AppDir。运行时、YXB、
锁定后端、资源、图标、许可证和摘要都已包含，目标机器无需安装言序/言包或联网。
各平台与 x86-64/ARM64 制品必须在对应原生 CI runner 构建和验证。
