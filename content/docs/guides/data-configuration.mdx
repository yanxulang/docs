---
title: 配置与数据
description: 使用言据建立可校验、可合并、可诊断的应用配置流程。
---

适合生产配置的顺序是“读取 → 解析 → 校验 → 合并 → 使用 → 规范写回”，每一步都保留结构化错误，不把失败降级成空值。

1. 从[言据格式](/ecosystem/yanju/format/)了解值模型、转义和规范序列化。
2. 用[校验与路径](/ecosystem/yanju/validation-paths/)声明字段、类型、范围与嵌套访问。
3. 用[数据处理与转换](/ecosystem/yanju/operations-conversion/)合并默认值、环境覆盖和用户配置。
4. 对连续记录使用[言据流](/ecosystem/yanju/streams-errors/)，并按错误代码处理失败。

配置文件所在目录需要在项目`[权限].文件`中声明；外部 JSON/TOML 可以作为互操作输入，但应用内部无需因此放弃言据的中文键名与规范输出。涉及密钥时，不要把完整值写入日志或错误消息。
