---
title: 项目清单
description: 阅读格式 2 的言序.toml，并区分必需表与可选工程边界。
---

下面是言包生成的常规最小清单形状：

```toml
[包]
格式 = 2
名称 = "问候"
版本 = "0.1.0"
言序 = ">=1.1.7"
入口 = "src/主.yx"

[依赖]

[权限]
文件 = []
网络 = []
本地网络 = false
TCP监听 = []
UDP绑定 = []
环境 = []
进程 = false
原生扩展 = false
图形界面 = false
剪贴板 = false
文件对话框 = false
系统通知 = false
托盘 = false
打开外部地址 = false
全局快捷键 = false

[导出]
默认 = "src/主.yx"

[构建]
目标 = "字节码"
```

格式 2 只强制要求`[包]`；言包模板显式写出安全默认值，方便审阅。可选表包括`[依赖]`、`[开发依赖]`、`[权限]`、`[导出]`、`[资源]`、`[构建]`、`[工作区]`和`[原生]`。清单不允许任意构建脚本或安装钩子。

资源目录写成：

```toml
[资源]
目录 = ["assets"]
```

使用`yanbao add/remove`编辑依赖，使用`yanbao install`生成锁图。字段与格式版本的集中速查见[项目格式参考](/reference/project-format/)。
