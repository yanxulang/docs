升级前先提交清单与锁文件，记录当前`yanxu version --json`，并在独立分支运行检查、格式化、规格测试、审计和构建。跨多个稳定版本时按时间顺序阅读；1.1.18 与 1.1.19 只有追溯快照，1.1.17 用户直接迁移到 1.1.20。
